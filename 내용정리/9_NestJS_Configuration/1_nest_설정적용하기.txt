이번에는 설정을 적용하는 방법을 조금 알아볼 것이다.

express 를 통한 웹 개발을 할 때는 보통 .env 파일을 활용하여 process.env.~~~ 이런 식으로 환경변수를 사용한다.
여기서 환경변수를 .env 파일에 따로 적는 이유는 코드에 남아있으면 안되는 보안이 중요한 정보이기 때문이다.


※  그럼 .env 파일에다 넣으면 100% 안전한가?
    .env 파일을 어떻게 관리하냐에 따라 다르다.
    prisma 를 사용할 경우 .env 파일이 알아서 만들어지는데
    그때 경고문을 보면 '.env 파일을 .gitignore 에 추가하는 것을 잊지 마십시오' 라고 적혀있다.
    즉, github 나 gitlab 등과 같은 곳에 올릴 경우 .env 파일은 반드시 .gitignore 에 추가하여
    다른 사람들이 해당 파일의 내용을 못 보게 해야한다는 것이다.


내가 작성한 코드들을 보면 process.env.SECRET_KEY 라는 것이 있다.
나는 prisma 에서 만들어주는 .env 파일에 추가해서 사용했다.
.env 파일에 SECRET_KEY="" 이런식으로 추가해주면
process.env 를 통해 해당 key 값에 접근할 수 있고 key 가 가지고 있는 value 값을
쓸 수 있다.


.env 파일을 통한 Configuration 말고도 
.yml 혹은 .json, .ts 등등 환경변수를 어떻게 관리하고 설정할 것인지는 방법이 여러가지인 것 같다.
여러가지 방법들 중에서 안전하다고 생각되는 방법을 통해 환경변수를 관리해주면 될 것 같다.
